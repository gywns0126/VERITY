# Dilution Rules v0.1

> VERITY ESTATE **내부 코어 → 외부 공개** 변환 규칙.
> 이 문서는 **내부 전용**.
> `scoring-landex.md`와 `scoring-gei.md`가 정의한 내부 데이터를 외부 채널로 내보낼 때의 필수 필터.

---

## 1. 원칙

1. **Core는 비공개, Output은 희석본만.**
2. **희석은 자동화 함수로 실행한다** — 매번 수작업으로 판단하면 실수로 유출된다.
3. **모든 공개 출력은 dilution filter를 반드시 통과**한다. 우회 경로 없음.
4. **변환 로그를 남긴다**. 어느 시점 내부 데이터에서 어떤 공개 출력이 나왔는지 역추적 가능해야 한다.
5. **과다 노출 자체가 유출이다.** 반복되면 내부 임계치·가중치를 역산할 수 있다.

---

## 2. 공개 허용 / 금지 전면 목록

### 2.1 ❌ 절대 공개 금지

| 항목 | 이유 |
|---|---|
| LANDEX 원점수 (예: 87.3) | 수치로 나가면 스코어 경계값·가중치 역산 가능 |
| GEI 원점수 (예: 47.2) | 동일 |
| 서브스코어 분해 (V·D·S·C·R / Rent·Merchant·…) | 카테고리별 비중 노출 |
| 가중치 (0.30·V 등) | 방법론 핵심 |
| 단계 배수표 (GTX 기획×0.3 등) | 호재 평가 로직 노출 |
| Stage 경계값 (40/65/85) | 판정 임계치 노출 |
| 피처별 기여도 | 어떤 피처가 어떤 방향으로 작동하는지 노출 |
| VAMS 포지션·PnL | 본인 투자 전략 유출 |
| AlertDashboard 알림 룰 | 감시 대상·조건 노출 |
| 과거 스코어 시계열 (구체 수치) | 백테스트 결과 역산 가능 |
| 백테스트 상관계수 | 모델 성능 정보 |

### 2.2 ⚠️ 조건부 공개

| 항목 | 조건 |
|---|---|
| 지역 랭킹 (25구 Top N) | 등급만 함께 공개. 숫자 순위 단독 공개 금지 |
| 과거 사례 언급 (경리단길 등) | 일반적 사회현상으로 설명 OK, 당시 GEI 수치는 비공개 |
| 데이터 소스명 | 카테고리 수준까지 (예: "공공 실거래 데이터"). 소스별 가중치는 비공개 |
| 방법론 카테고리 (가치·수요·공급·호재·리스크) | 이름까지만 OK. 구성 피처·비중 비공개 |

### 2.3 ✅ 공개 허용

| 항목 | 허용 사유 |
|---|---|
| LANDEX 5등급 라벨 (HOT/WARM/NEUTRAL/COOL/AVOID) | 의도적 요약 정보 |
| GEI Stage 라벨 (초기/가속/정점/조정) | 동일 |
| 조기경보 이벤트 (이번 주 A구 가속 신호) | 브랜딩 자산. 단, 발행 빈도 제한 |
| 지역 일반 묘사 (상권 동향·호재 뉴스) | 공공 정보 재구성 |
| 매크로 해설 (금리·전세가율 전국 추이) | 공공 데이터 해석 |

## 3. 변환 함수 스펙

### 3.1 LANDEX 변환

```python
def dilute_landex(score: float, gu: str) -> PublicLandex:
    label = {
        (80, 101): "HOT ZONE",
        (65,  80): "WARM",
        (45,  65): "NEUTRAL",
        (30,  45): "COOL",
        ( 0,  30): "AVOID",
    }[bucket(score)]

    return PublicLandex(
        gu=gu,
        label=label,              # ✅ 공개
        comment_tone=COPY[label], # ✅ 공개 (방향성만)
        # 아래는 모두 출력에 포함 안 함
        # raw_score, subscores, features, weights
    )
```

출력 예:
> "성동구: WARM — 주목할 흐름"

절대 이렇게 나가면 안 됨:
> ❌ "성동구: 72점 (V 78, D 65, C 81, R 2점 차감)"

### 3.2 GEI 변환

```python
def dilute_gei(gei: float, gu: str) -> PublicGei | None:
    if gei < 20:
        return None   # Stage 0은 언급 자체 안 함

    stage = {
        (20, 40): "관심 구간",
        (40, 65): "가속 신호",
        (65, 85): "과열 구간",
        (85, 101): "조정 신호",
    }[bucket(gei)]

    return PublicGei(gu=gu, stage=stage, comment_tone=COPY[stage])
```

## 4. 채널별 희석 강도

| 채널 | 희석 강도 | 허용 상세도 |
|---|---|---|
| 인스타 카드뉴스 | **최대** | 라벨 + 1문장 코멘트 |
| 블로그·뉴스레터 | 중간 | 방향성 설명 + 공공 데이터 인용 + 방법론 카테고리명 |
| 유튜브·오디오 | **최대** | 인스타와 동급 (말로 유출되기 쉬워 오히려 더 타이트) |
| 비공개 링크 공유 (친구·소수) | 낮음 | Stage·등급 + 근거 설명. 단 **원점수·가중치는 여전히 비공개** |
| 본인 Notion/Obsidian (내부) | 없음 | 풀디테일 |

**비공개 링크 공유 주의**:
"반공개 등급"을 만들면 누수 지점이 늘어난다. 친한 사람에게도 원점수·방법론 디테일은 공유하지 않는다. 등급·방향성까지만.

## 5. 발행 정책

### 5.1 주기 제한

| 채널 | 최대 발행 빈도 |
|---|---|
| 인스타 카드뉴스 | 주 1회 (정기) + 이벤트성 월 최대 2회 |
| 뉴스레터·블로그 | 월 1회 (정기 리포트) |
| GEI 조기경보 긴급 공개 | 월 최대 2회 |

**이유**: 노출 빈도 자체가 희석의 한 축이다. 매일 발행하면 내부 스코어 변동 패턴이 드러난다.

### 5.2 발행 전 체크리스트

공개 전 **자동으로** 검증:

- [ ] dilution filter 통과 (원점수·수치 없음 확인)
- [ ] combination guard 통과 (같은 구 LANDEX+GEI 동시 노출 아님)
- [ ] 지역 중복 노출 간격(최소 4주) 확인
- [ ] 데이터 소스 "카테고리 수준"까지만 언급
- [ ] VAMS 포지션·알림 룰 참조 없음
- [ ] 가중치·임계치·단계 배수표 누출 없음

체크리스트 미통과 시 자동 보류. 수동 발행 금지.

## 6. 로그·감사

### 6.1 변환 로그 (필수)

모든 공개 출력에 대해 기록:

```json
{
  "post_id": "ig-2026-04-23-001",
  "channel": "instagram",
  "published_at": "2026-04-23T10:00:00+09:00",
  "internal_refs": {
    "landex_snapshot_version": "2026-04-22-monthly",
    "gei_snapshot_version": "2026-04-22-weekly",
    "source_gu": ["성동구"],
    "source_scores": "encrypted-blob"  // 감사용, 공개 경로에 접근 불가
  },
  "public_content": {
    "label": "WARM",
    "comment": "..."
  },
  "dilution_filter_pass": true,
  "combination_guard_pass": true
}
```

### 6.2 사후 감사

월 1회:
- 최근 4주간 공개 출력 전수 점검
- 같은 구 반복 노출 빈도 확인
- 라벨 분포 편향 확인 (WARM만 계속 나가면 경계값 추론 가능)
- 이상 발견 시 변환 함수 수정

## 7. 실수 유출 대응

### 7.1 유출 등급

| 등급 | 정의 | 대응 |
|---|---|---|
| **L1: 경미** | 방향성 수준 초과 노출 (예: "WARM 상위권") | 해당 포스트 수정 |
| **L2: 중대** | 서브스코어·피처 기여도 노출 | 포스트 삭제 + 다음 발행 2주 중지 |
| **L3: 치명** | 원점수·가중치·임계치 노출 | 포스트 삭제 + 해당 채널 1개월 중지 + 내부 감사 |

### 7.2 감사 기록

L2·L3 사건은 `docs/incidents/` 폴더에 사건 ID·원인·재발 방지 조치 기록.
