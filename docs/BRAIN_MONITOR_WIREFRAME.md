**KPI 4개 카드 (우상단)**:
- **Brain Health Score** (0~100, 어제 대비 ±N)

## 7. 3D Brain — 노드 구조

**좌측 클러스터 (Input Sources, 5 nodes)**:
- 가격 (price) — yfinance / KIS
- 재무 (financials) — DART / SEC
- 매크로 (macro) — FRED / ECOS
- 뉴스 (news) — RSS / sentiment
- 섹터 (sector) — sector data

**중앙 클러스터 (Engines, 4 nodes)**:
- Fact Engine (multi_factor, consensus, prediction, backtest 등)
- Sentiment Engine (news_sentiment, x_sentiment, market_mood)
- Risk Filter (red_flags, macro_override, panic_stage)
- VCI Comparator (fact-sentiment 괴리 측정)

**우측 클러스터 (Output, 3 nodes)**:
- Brain Score (0~100)
- Grade (STRONG_BUY/BUY/WATCH/CAUTION/AVOID)
- Confidence (0~1)

**연결선**: 데이터 흐름. 두께 = 기여도, 색 = 정상/이상.
**노드 색**: 🟢 정상 (95%+ 성공률) / 🟡 경계 (90~95%) / 🔴 장애 (<90%).

**인터랙션**:
- 노드 클릭 → 우측 상세 패널 (해당 소스 7일 추이 + 알림 + 로그 링크)
- 마우스 호버 → 툴팁 (현재 값 + 어제 대비 변화)
- 노드 더블클릭 → 해당 탭으로 이동 (예: 매크로 노드 → Data Health 탭의 매크로 소스)

## 8. 화면 외 알림 — Telegram 통합

기존 텔레그램 봇과 통합:
- 🔴 장애 발생 → 즉시 텔레그램 푸시 + 알림 패널 추가
- 🟡 경계 → 1시간 누적 후 푸시
- 🟢 복귀 → 조용히 알림 패널만

알림 메시지 예시:
```
[VERITY] 🔴 macro_data 파싱 실패
시각: 14:23 KST
영향: 매크로 노드 → Brain Score 신뢰도 ↓
액션: 14:30 cron 재시도 또는 수동 트리거
링크: brain.verity.app/data-health
```

## 9. 디자인 원칙

1. **Bloomberg 터미널 톤** — 검정 배경, 네온 액센트, mono 숫자, 한국어/영어 혼용 (영문은 약자 위주)
2. **장식 최소** — 그림자/그라데이션 거의 없음, 정보 밀도 높게
3. **색은 신호로만** — 🟢🟡🔴 외 임의 색 금지 (브랜드 액센트 #B5FF19 제외)
4. **숫자는 mono 폰트 + 정렬** — 비교 가능성
5. **3D는 보조** — 메인 정보는 2D 차트/표, 3D는 "전체 상태 한눈에"
