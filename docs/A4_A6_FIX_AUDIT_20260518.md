## §1 — A6 commodity_margin 진단 + fix

### 코드 위치
- `api/collectors/CommodityScout.py:527` `attach_commodity_to_stocks(stocks, scout)` 호출자 = `api/main.py:3068`
- audit doc § A 박힘 — "stock['commodity_margin'] 박는 collector 호출 흔적 없음" 가설은 **부분 오류**. 실제 attach_commodity_to_stocks 가 호출됨.

### Root cause (옛 logic)
```python
t = str(s.get("ticker", "")).zfill(6)
s["commodity_margin"] = by_t.get(t, {})
```
- KR `"005930"` → `"005930"` (정상)
- **US `"TMO"` → `"000TMO"` (zero padding 버그)** ← 핵심
- US `"AAPL"` → `"00AAPL"`
- → US 15 종목 모두 매칭 fail
- KR 도 by_ticker 자체 빈 시 0 (별도 결함, §2 참조)

### Fix (Tier 1)
```python
raw = str(s.get("ticker", ""))
t6 = raw.zfill(6) if raw.isdigit() else raw  # KR 만 padding
s["commodity_margin"] = by_t.get(raw) or by_t.get(t6) or {}  # raw + t6 둘 다 시도
```
- KR `"005930"` → raw="005930" 또는 t6="005930" 매칭 OK
- US `"TMO"` → raw="TMO" 매칭 OK

### 부분 해소 — by_ticker 자체 비어있는 결함 (다음 sprint)

`portfolio.commodity_impact = {}` 빈 (5/17 14:21 baseline 시점).
- 가능 1: VAMS reset run = quick or reset mode = STEP 5.8 commodity scout 실행 X (정상 baseline)
- 가능 2: full mode 실행 됐지만 run_commodity_scout 결과 빈 dict 반환 (`api/collectors/CommodityScout.py:422` 정독 필요)

→ 다음 sprint audit 큐잉 (16:07 KST daily_full 후 portfolio.commodity_impact 채워졌는지 확인 → 빈이면 run_commodity_scout 진단)

## §2 — A4 sec_financials 누락 5 US 진단 + fix

### 누락 5 종목 (sanity 측정 5/18 12:55 KST)
**TMO / PG / XOM / CSCO / JNJ** — 모두 mega-cap, sector 다양 (healthcare / consumer staples / energy / tech / pharma).
채워진 10 종목: SOFI / QCOM / DIS / CRM / BAC / ADBE / JPM / MSFT / META / BRK-B

### Root cause (옛 logic, api/main.py:2449-2459)
```python
if not stock.get("sec_filings") or not stock.get("sec_financials"):
    try:
        from api.collectors import sec_edgar as _sec
        from api.config import SEC_EDGAR_USER_AGENT as _ua
        if _ua:
            ...
            stock["sec_financials"] = _sec.get_financial_facts(ticker, _ua)
    except Exception:
        pass  # ← silent skip, feedback_data_collection_verification_mandatory 위반
```

옛 logic `except Exception: pass` = silent skip. 5 종목 fail 시 trace 0건. root cause 진단 불가.

### Fix (Tier 1, silent skip 보강)
- `SEC_EDGAR_USER_AGENT` env 부재 시 stderr 명시
- `except Exception as _e:` 로 변경 + `_e.__name__ + str(_e)[:200]` stderr 박음
- next cron run trace 확보 → root cause 식별 가능 → 본격 fix 다음 sprint

### 가설 (다음 sprint 진단 의제)
- 가설 a: SEC EDGAR API 의 ticker→CIK 매핑 cache stale (특정 ticker 만 회귀)
- 가설 b: SEC EDGAR rate limit (10 req/sec 한도, 15 동시 fetch 시 일부 throttle)
- 가설 c: ticker symbol parser 결함 (BRK-B 같은 dash ticker 처리 가능, TMO 등 단순 ticker 결함 패턴 = 의심)
- 가설 d: get_financial_facts 함수 자체 결함 (특정 company facts schema 변경 후 회귀)

trace 확보 후 진단 → 가설 a/b/c/d 중 1 또는 2 식별 → 본격 fix.
